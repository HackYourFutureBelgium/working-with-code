Creating a Markdown project


  1. Create a new folder on your computer
  2. Open it in VSC
  3. Add a LICENSE
  4. Create your README.md & preview it
  5. Title & intro of your README.md
  6. Create a table of contents
  7. Write more files (linked to the README)

Proofreading like a dev


Writing code is still writing!
Take the time to proof read and edit your code. Your colleagues and your future self will thank you.

  • Correct your spelling with Code Spell Checker
  • Format your code with Prettier
  • Fix any linting errors
  • Preview the rendered Markdown, does it look like you wanted?

Group Intro Repository, Pt. 1


Writing The Code


(group exercise)

What you will make


Each of you will create a new folder on your computer with short bio for each person in your small group. It should have ...

A LICENSE file


The MIT License is a good one

A main file called README.md


This file includes:

  • Your group name (be creative ;)
  • A nice introduction to your group
  • A list of links to all the other files

One file per group member


Take turns introducing yourselves!

Create a new file for each member of your group called their-name.md. Write a short introduction for them in this file.

By the end you should one file for each member in your group
